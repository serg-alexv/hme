# LEGION Roadmap

## Operating principle

The roadmap is gate-driven, not calendar-fantasy-driven. A later phase cannot borrow the status of an earlier unverified phase.

## Milestone 0 — Repository control plane

**Objective:** durable authorship, governance, status, claims, agent roles, and task interfaces.

**Exit criteria:**

- [x] README and repository map
- [x] governance and Y/N mode
- [x] agent swarm roles
- [x] status ledger
- [x] claims register
- [x] canonical architecture
- [ ] rename repository from staging name `hme` to `legion`
- [ ] import preserved source packages and operator freeze through a reviewed migration

## Milestone 1 — Causal desktop proof

**Objective:** make the causal link obvious without explanation.

```text
input metric changes
→ feature values change
→ state changes
→ dragon motion changes
```

**Required deliverables:**

- Lens Studio TCP listener confirmed;
- test frame updates target component;
- live/replay sender drives all four states;
- visible overlay shows source, RSSI/strength, variability, novelty, state, freshness, sequence;
- stopping sender produces stale behavior;
- complete experiment bundle frozen and checksummed.

**Exit condition:** a technically literate viewer understands the causal chain within ten seconds.

## Milestone 2 — Portable mobile proof

**Objective:** prove the same real trace can drive a phone-visible Snap Lens without claiming live transport.

**Deliverables:**

- recorded trace embedded as a Lens asset;
- deterministic replay controller;
- provenance label `RECORDED_REPLAY`;
- phone recording;
- package manifest and exact Lens version.

## Milestone 3 — First paid external proof

**Objective:** obtain one payment for a bounded deliverable.

**Candidate offers:**

- reactive-character architecture audit;
- sensor-to-creature prototype sprint;
- branded demonstrator;
- museum/STEM workshop pilot.

**Exit criteria:**

- signed scope;
- deposit received;
- deliverable accepted;
- case-study evidence preserved;
- claims unchanged or strengthened by evidence.

## Milestone 4 — Live mobile branch decision

Y/N decision between:

### Route A — Camera Kit

Choose when Snap distribution and Lens reuse dominate.

### Route B — Native iOS

Choose when sensor access, BLE, persistence, background behavior, and sovereignty dominate.

The decision must record platform access, cost, expected users, required sensors, and rollback.

## Milestone 5 — Scientific moat

Activate only with partner, funding, or demonstrated product demand.

Possible adapters:

- magnetometer;
- ESP32 CSI;
- multi-node RF/motion sensing;
- calibrated external field sensors.

**Required discipline:** independent validation, calibration, error metrics, subject/device generalization, false positives, raw-data preservation.

## Milestone 6 — Persistent world

Activate only after multi-user product behavior exists.

Possible capabilities:

- persistent creatures;
- trusted measurement events;
- campaigns and rewards;
- global relationships and territory;
- anti-fraud and idempotent scoring.

Database selection follows workload evidence. Spanner is an option, not a premise.

## Deferred until activation gates

- ARCADIA runtime integration;
- ADK in the fast control loop;
- Pub/Sub for a single producer/consumer;
- Worker Pools without continuous backlog;
- cold-storage optimization for tiny active traces;
- large multi-year strategic funding asks before a paid proof.
